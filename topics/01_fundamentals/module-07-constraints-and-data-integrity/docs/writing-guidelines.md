# Writing Guidelines (Module)

Pedoman penulisan module ini:

1. Jelaskan constraint sebagai aturan kualitas data, bukan hanya syntax.
2. Bedakan fungsi tiap constraint dengan bahasa sederhana.
3. Gunakan contoh SQL pendek dan relevan untuk pemula.
4. Sertakan diagram atau visual ringkas bila membantu membandingkan constraint.
5. Hindari pembahasan relasi lanjutan atau behavior lanjutan yang terlalu dalam.
6. Tambahkan kesalahan umum pemula saat mulai memakai constraint.
