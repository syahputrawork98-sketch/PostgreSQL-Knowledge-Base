# Writing Guidelines (Module)

Pedoman penulisan module ini:

1. Utamakan contoh SQL yang pendek, jelas, dan bertahap.
2. Jelaskan hubungan antara setiap query dan perubahan data yang terjadi.
3. Beri perhatian khusus pada risiko `UPDATE` dan `DELETE` tanpa `WHERE`.
4. Gunakan bahasa yang praktis dan langsung ke penggunaan sehari-hari.
5. Tambahkan kesalahan umum pemula saat mulai memanipulasi data.
