# Workflow Project

Untuk memastikan kualitas dan keteraturan, pengembangan project ini mengikuti alur kerja berikut:

1. **DISCUSS**: Mendiskusikan kebutuhan atau ide fitur baru.
2. **ANALYZE**: Menganalisis teknis dan dampak perubahan.
3. **DECIDE**: Mengambil keputusan akhir mengenai apa yang akan dikerjakan.
4. **EXECUTE**: Melakukan implementasi kode atau penulisan materi.
5. **REPORT**: Melaporkan hasil pekerjaan.
6. **COMMIT/PUSH**: Menyimpan perubahan ke repository (Source of Truth).
7. **REVIEW**: Meninjau hasil pekerjaan bersama owner/manager.
8. **NEXT STEP**: Menentukan langkah selanjutnya untuk batch berikutnya.
